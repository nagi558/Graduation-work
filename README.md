## 8. このアプリの懸念点とその対策
※最低1つは必ず書いてください。

このアプリが**うまくいかない可能性**と、
それに対して**どう向き合おうとしているか**を書いてください。

### 懸念点①

- 何が問題になりそうか：ログイン機能
- なぜそう思うか：ログインが面倒くさいので、気軽さがなくなってしまう。
- そのために考えている対策：ログインはアドレスとパスワードのみのシンプルなものにする。ログインをしなくても使える機能を増やす。

### 懸念点②

- 何が問題になりそうか：カテゴリ分け
- なぜそう思うか：カテゴリが増えすぎてしまうことで、煩雑になってしまう。逆に少なすぎると探しにくくなってしまう。
- そのために考えている対策：大項目、小項目に分け、後からカスタマイズで追加できるように検討する。

### 10-2. 技術選定の理由

- なぜこの技術を使うのか
1.ユーザーがさくさく書けることを意識したいので、Next.jsを使用したい。
2.MVPをまずは行いことが大事だと思うので、Railsを混ぜずにNext.jsを使用してスピード感を意識してみたい。
3.今回はメモストックアプリのため、データが命なので、安全に保存ができるという面でPostgreSQLを選択
4.TypeScriptを使用するので、型安全性を考えてPrismaを選択
5.若者向けかつ子ども関係のアプリなので、くすみカラーのようなカラーにこだわりたい。そこで、UIのカスタマイズ性が高いと評判のshadcn/uiを使用する。
- 今回チャレンジしたい点
アプリの安全性を考慮すると、Railsと掛け合わせた方がより確実になるとのことですが、今回はMVPまでのスピード感を意識したいので、Next.jsを使用します。
しかし、アプリの発展度合いによっては、今後Railsをバックエンドとして使用する必要もあると考えています。AIに聞いたり、Googleで調べた範囲ですが、
Server ActionsからService層を切り離して書くことを意識することで、後々のバックエンドの切り離しをスムーズにしたいと考えています。
- 不安な点
Next.jsは学び始めたばかりなので、インプットが間に合うかが一番不安です。
ただし、調べながらインプットと並行してアウトプットをしていきたいと思っています。
